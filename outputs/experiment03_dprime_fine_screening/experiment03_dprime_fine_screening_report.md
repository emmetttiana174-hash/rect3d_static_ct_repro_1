# 三维静态 CT 光源展布实验 3：d' 任务精筛实验

记录日期：2026-05-26

## 1. 实验目的

实验二已经证明 EIG 可以在大量三维候选光源架构中形成有意义的粗筛排序，但 EIG 本身仍是平均信息量指标，并不直接等价于具体成像任务的最优可探测性。本实验的目的，是在 EIG top 候选池内部继续引入任务驱动的 d' 精筛，检验高信息架构之间是否仍存在任务性能差异。

本组实验要验证的核心命题是：在环形探测器静态 CT 中，系统最优不应只由全局参数精度或平均信息量定义，而应由具体成像任务定义。因此，EIG 粗筛之后仍需要 d' 精筛。

## 2. 候选来源

本实验不再面对实验二中的全部 240 个候选架构，而是只使用实验二保留下来的 EIG top 10% 候选，共 24 个架构。这样做对应 EIG+d' 分层设计路线：

```text
大量参数化候选架构 -> EIG 粗筛 -> 少量高信息候选 -> d' 任务精筛
```

因此，本实验关注的不是 EIG 能否排除低信息架构，而是：在已经通过 EIG 初筛的架构中，哪一个更适合具体任务。

## 3. 任务定义

| task | 含义 |
|---|---|
| `center_small_target` | 中心小球形低对比目标检测 |
| `eccentric_roi_target` | 偏心 ROI 小球形低对比目标检测 |
| `material_contrast` | 局部材料/低对比差异判别任务 |

其中前两个任务是局部小目标检测，分别对应中心 ROI 和偏心 ROI；第三个任务是局部材料/低对比差异判别，用来避免精筛准则只服务于单一检测位置。

## 4. d' 近似计算方式

当前 quick 实验没有显式构造完整三维 Fisher 矩阵或完整 MAP Hessian，因为 32 x 32 x 32 体素下完整矩阵规模已经较大。这里采用低维通道化 MAP observer 近似：先围绕每个任务位置构造局部高斯通道，再把候选架构的投影 Fisher 信息投影到通道空间，计算任务信号在该通道空间中的可分离度。

计算形式可以理解为：

```text
d_t'^2(Theta) = Delta_mu_c,t^T K_c,t^{-1} Delta_mu_c,t
```

其中 `Delta_mu_c,t` 是任务信号在通道空间中的线性响应，`K_c,t` 是通道空间中的噪声/不确定性矩阵。当前实现加入 MAP 正则项 `beta`，避免 quick 近似下矩阵病态。

- 通道数：`12`
- MAP 正则参数 beta：`2.5e+04`
- 光子预算、探测器采样、总源数与实验二保持一致。

## 5. 精筛准则

本实验采用 maximin 任务准则：

```text
U_task(Theta) = min_t d_t'^2(Theta)
```

也就是说，最终选择的不是只在某一个任务上特别强的架构，而是在最弱任务上仍然尽可能好的架构。这个准则更符合多任务安检 CT 的设计目标，因为真实系统不能只对一个固定位置或一种目标表现好。

需要特别说明：本实验没有对三个任务设置加权求和，也不是要求最终候选在三个任务上都分别达到单项最优。`maximin` 准则关注的是每个候选架构的“短板任务”。例如某个架构在中心目标和材料判别上很强，但在偏心 ROI 目标上明显较弱，那么它的最终任务效用仍然会被偏心 ROI 任务限制。

因此，`cand_0236` 被选中并不是因为它在三个任务上都第一，也不是因为三个任务加权平均后最高，而是因为：

```text
min(d'_center, d'_eccentric_roi, d'_material)
```

在所有 EIG top 候选中最大。换句话说，它是当前三任务下最稳健的候选。

## 7. EIG 排名与 d' 排名的关系

EIG 排名第 1 的候选为 `cand_0216`，其 d' 精筛排名为 `22`，maximin min d' 为 `132.186`。
d' 精筛最终选中的候选为 `cand_0236`，其 EIG 排名为 `7`，maximin min d' 为 `138.407`。

这说明 EIG 最优架构并不等于任务最优架构。EIG 负责筛出平均信息价值较高的候选，但在这些候选内部，具体中心目标、偏心 ROI 目标和材料判别任务的 d' 排名仍会重新排序。

因此，本实验支持 EIG+d' 的分层逻辑：EIG 用于快速删除低信息价值架构，d' 用于在高信息候选中选择更符合具体任务需求的架构。

## 8. 输出图像含义与读图方式

### 8.1 `fig01_eig_rank_vs_dprime_rank.png`

这张图用于比较 EIG 粗筛排序和 d' 任务精筛排序之间是否一致。它包含两个子图。

左侧子图的横轴是候选在 EIG top 24 内部的排名，横轴越靠左表示 EIG 越高；纵轴是 d' maximin 排名，纵轴越靠上表示任务精筛表现越好。图中的灰色虚线表示“EIG 排名与 d' 排名完全一致”的理想情况。如果所有点都落在虚线附近，说明 EIG 排名基本可以替代 d' 排名；如果点明显偏离虚线，说明 EIG 和任务 d' 给出的排序不同。

当前结果中，最关键的点是：

```text
EIG rank 1: cand_0216 -> d' rank 22
d' rank 1: cand_0236 -> EIG rank 7
```

这说明 EIG 最高的候选并不是任务 maximin 最优候选。也就是说，EIG 能把低信息架构筛掉，但在高 EIG 候选池内部，仍然需要 d' 根据具体任务重新排序。

右侧子图把候选按 EIG 排名从高到低排列，纵轴画的是每个候选的 `min task d'`，也就是三个任务里最差任务的 d'。这张子图的读法是：看 EIG 排名前列的候选，其最弱任务性能是否也一定最高。当前并不是这样。`cand_0216` 虽然 EIG 最高，但它的最弱任务是偏心 ROI 任务，min d' 只有 `132.186`；而 `cand_0236` 的 min d' 为 `138.407`，因此被 d' 精筛选中。

因此，图一要表达的核心信息是：

```text
EIG 排名和任务 d' 排名相关但不能互相替代；
EIG top 候选内部仍存在明显任务适配差异。
```

### 8.2 `fig02_task_dprime_bars.png`

这张图展示 d' 精筛排名靠前的候选在三个任务上的具体 d' 数值。横轴是候选架构，按 d' maximin 排名排序；每个候选有三根柱子，分别对应：

| 柱子 | 含义 |
|---|---|
| `center_small_target` | 中心小目标检测 d' |
| `eccentric_roi_target` | 偏心 ROI 小目标检测 d' |
| `material_contrast` | 材料/低对比判别 d' |

读这张图时，不能只看最高的柱子，而要看每个候选中最低的那根柱子。因为本实验采用 maximin 准则，最终排名由“短板任务”决定。

当前图中可以看到，`material_contrast` 的 d' 通常明显高于另外两个检测任务，因此它不是大多数候选的限制因素。真正决定 maximin 排名的，主要是 `eccentric_roi_target`，也就是偏心 ROI 小目标检测任务。`cand_0236` 的偏心 ROI d' 为 `138.407`，在当前候选中最高，同时中心任务 d' 为 `146.320`，材料任务 d' 为 `222.357`，没有明显失衡，因此成为最终候选。

相比之下，`cand_0216` 在中心任务上达到 `147.954`，是中心任务单项最优；在材料任务上也达到 `231.746`，表现很强。但它的偏心 ROI d' 只有 `132.186`，这个短板使它不适合作为三任务稳健最优架构。

因此，图二要表达的核心信息是：

```text
d' 精筛不是寻找某一根柱子最高的候选，
而是寻找三项任务中最低柱子尽可能高的候选。
```

这也解释了为什么 `cand_0236` 是最终候选：它不是所有任务都单项第一，而是在当前任务集合中最均衡、最稳健。

## 10. 阶段性结论

实验三的关键作用，是把实验二的“高信息架构筛选”推进到“具体任务可探测性筛选”。当前 quick 结果应作为方法链验证：它证明了 EIG top 候选之间仍需要根据任务 d' 进行比较，并给出了 maximin 任务准则下的最终候选。

最重要的论文式表述是：EIG 最优架构不一定是任务最优架构；在通过 EIG 粗筛后，仍需使用 d' 进行任务驱动精筛。即使在某些 quick 参数下两者恰好一致，d' 精筛仍提供了必要的任务性能验证。

# 三维静态 CT 光源展布实验 2：EIG 粗筛实验

记录日期：2026-05-24

补充说明日期：2026-05-25

## 1. 实验目的

本实验用于验证 EIG 是否适合作为三维光源架构搜索的第一层粗筛准则。它不直接回答最终检测任务最优，而是检验 EIG 是否能在大量参数化候选架构中形成有意义的排序，并把明显低信息价值的架构排除掉。

如果 EIG 排序没有结构性偏好，或者 top 候选与全体候选的几何参数分布没有差异，那么后续 d' 精筛就缺少可靠前提。因此，本实验是 EIG+d' 分层设计路线中的第一层创新证据。

## 2. 候选架构空间

候选架构不是任意三维点集，而是在参数化多层环架构族中采样：

```text
Theta = {M_z, N_m, R_m, z_m, delta_m, alpha_m}
```

| 参数 | 含义 | 当前 quick 范围 |
|---|---|---|
| `M_z` | 层数 | 1, 2, 3, 4 |
| `N_m` | 每层源数 | 总源数固定为 48，按层数分配 |
| `R_m` | 源环半径 | 1.25 到 1.70 |
| `z_m` | 最大层高半幅 | 0.15 到 0.60，单层为 0 |
| `delta_m` | 层间角偏移比例 | 0 到 1 |
| `alpha_m` | 朝向插值参数 | 0 为中心定向，1 为 ROI 定向 |

本次使用 Latin hypercube 采样 `240` 个候选架构，并保留 EIG 排名前 `24` 个候选，约为 top `10.0%`。

## 3. 对象先验库

EIG 使用三类对角高斯先验对象：

| 先验类别 | 权重 | 含义 |
|---|---:|---|
| `smooth_background` | 0.350 | 平滑背景对象 |
| `center_weak_target` | 0.325 | 平滑背景 + 中心弱目标 |
| `eccentric_roi_target` | 0.325 | 平滑背景 + 偏心 ROI 弱目标 |

## 4. EIG 近似计算方式

理论目标是对每个先验类计算 Laplace EIG：

```text
U_EIG,c(Theta) = 1/2 logdet(I + Sigma_c^(1/2) F_c(Theta) Sigma_c^(1/2))
```

三维全矩阵 logdet 代价很高，因此当前 quick 实验使用固定随机低维探针子空间近似该 logdet。具体做法是用 ASTRA `cone_vec` 对 covariance-weighted probe volume 做 forward projection，形成低秩矩阵 `B`，再计算：

```text
U_EIG,c approx 1/2 logdet(I + B^T B)
```

多类先验按权重加权：

```text
U_EIG_coarse = sum_c pi_c U_EIG,c
```

## 7. 输出图像含义与结果分析

### 7.1 `fig01_eig_distribution.png` 的含义

`fig01_eig_distribution.png` 是全部候选架构的 weighted EIG 分布图。横轴表示每个候选三维光源架构的加权 EIG 数值，纵轴表示落入对应 EIG 区间的候选数量。图中的分布不是用来判断最终检测性能，而是用来回答一个更基础的问题：

```text
EIG 是否能把 240 个参数化候选架构区分开，并形成可用于粗筛的排序？
```

当前实验中，全部 `240` 个候选的 weighted EIG 统计为：

| 指标 | 数值 |
|---|---:|
| 最小值 | 3.08294 |
| 均值 | 3.27307 |
| 中位数 | 3.26063 |
| 75% 分位数 | 3.35038 |
| 最大值 | 3.57964 |
| 标准差 | 0.10882 |
| top 10% 阈值 | 3.42751 |
| top 10% 均值 | 3.47198 |

从这些数值可以看出，EIG 并不是所有架构都给出近似相同的评价。最大值 `3.57964` 相比最小值 `3.08294` 高约 `16.11%`，top 10% 候选的均值 `3.47198` 相比全体均值 `3.27307` 高约 `6.08%`。这说明当前三维参数化架构空间中确实存在信息价值差异，EIG 可以作为第一层粗筛指标，把明显低信息价值的候选架构排除掉。

需要注意的是，`fig01_eig_distribution.png` 不能直接说明“EIG 最高的架构就是最终检测最优架构”。它只能说明：在当前先验和 Laplace 近似下，某些架构对先验不确定性的平均信息压缩能力更强。因此，这张图支持的是“EIG 可以排序和缩小搜索空间”，而不是“EIG 已经完成最终任务优化”。

fig01_eig_distribution.png 的作用是看这 240 个 EIG 分数是不是有明显差异。如果所有分数都挤在一起，说明 EIG 没有筛选能力；但现在它们有可见差异，所以 EIG 可以作为粗筛工具。
### 7.2 `fig02_top_parameter_distributions.png` 的含义

`fig02_top_parameter_distributions.png` 比较的是全部候选架构与 EIG top 10% 候选架构的几何参数分布。它要回答的问题是：

```text
EIG 选出的高信息架构是否具有可解释的几何偏好？
```

如果 top 候选与全部候选在层数、半径、层高、角偏移和朝向参数上几乎没有差异，那么 EIG 粗筛虽然给出了数值排序，但未必揭示了有意义的架构规律。当前结果则相反，top 候选呈现出比较清楚的几何偏好。

首先看层数分布：

| layer_count | 全部候选比例 | EIG top 10% 比例 |
|---:|---:|---:|
| 1 | 0.250 | 0.000 |
| 2 | 0.250 | 0.583 |
| 3 | 0.250 | 0.292 |
| 4 | 0.250 | 0.125 |

全部候选中 1、2、3、4 层各占 25%，但 EIG top 10% 中没有任何单层架构，且 2 层和 3 层合计占 `87.5%`。这说明在当前 3D 先验和探测器设置下，EIG 明确不偏好单层环；轴向分层对提高三维信息获取能力是重要的。

再看连续几何参数：

| 参数 | 全部候选均值 | EIG top 10% 均值 | 变化趋势 |
|---|---:|---:|---|
| `sources_per_layer_mean` | 25.0000 | 20.1667 | top 候选更偏好多层，因此每层源数更少 |
| `radius` | 1.4750 | 1.3024 | top 候选偏向较小源环半径 |
| `z_extent` | 0.2844 | 0.4628 | top 候选明显偏向较大轴向层高 |
| `angular_offset_fraction` | 0.5000 | 0.4378 | top 候选没有强烈固定偏移偏好 |
| `orientation_alpha` | 0.5000 | 0.4866 | top 候选没有明显偏向全中心或全 ROI 定向 |

其中最重要的趋势是 `z_extent`：top 10% 的平均层高半范围为 `0.4628`，比全体候选均值 `0.2844` 高约 `62.75%`。这说明 EIG 粗筛明显偏好具有更大轴向覆盖的多层架构。结合实验一中双层固定规则架构出现 upper slab RMSE 异常的问题，这一结果说明后续优化不应简单固定双层高度，而应让 EIG / d' 在较宽的 `z_extent` 范围内自动选择。

`radius` 的变化也值得注意：top 10% 的平均源环半径为 `1.3024`，低于全体均值 `1.4750`，下降约 `11.70%`。在当前 cone angle 和 detector tile 设置下，较小半径可能让射线束对有限体积和先验高方差区域形成更有效覆盖。不过这一点仍需要后续 d' 精筛和泊松重建验证，因为过小半径也可能带来几何放大、截断或视角覆盖方面的副作用。

`orientation_alpha` 的 top 均值为 `0.4866`，接近全体均值 `0.5000`，说明 EIG 粗筛本身并没有强烈偏向“全中心定向”或“全 ROI 定向”。这与实验一的结果是一致的：全 ROI 定向并没有直接改善 ROI RMSE，因此朝向参数不应在规则基线阶段被固定到极端值，而应该留给后续 d' 精筛根据具体检测任务进一步判断。

fig02_top_parameter_distributions.png 的作用是看“高分架构”有什么共同特征。当前结果发现：

EIG 高的架构基本不是单层环；
更偏好多层；
更偏好较大的 z 方向层高；
更偏好较小的源环半径；
但不明显偏向全中心定向或全 ROI 定向。
### 7.3 两张图共同说明的关键信息

两张图合在一起说明：EIG 粗筛并不是任意筛选，而是在当前三维参数化候选空间中形成了有结构的几何偏好。

具体来说：

1. `fig01_eig_distribution.png` 说明 weighted EIG 在候选之间存在可观测差异，top 10% 的 EIG 明显高于总体均值；
2. `fig02_top_parameter_distributions.png` 说明这些高 EIG 候选不是随机分布的，而是明显偏好多层、大 `z_extent` 和较小 `radius`；
3. 单层架构在 top 10% 中完全消失，说明三维问题中轴向分层是 EIG 认为重要的自由度；
4. `orientation_alpha` 没有明显单边偏好，说明 EIG 粗筛阶段不应直接替代任务驱动的 d' 精筛；
5. 后续实验应把 EIG top 5% 到 10% 作为候选池，再用 d' 针对中心目标、偏心 ROI 目标或其他局部任务进行精筛。

因此，实验二的直接结论是：EIG 适合作为三维光源展布优化的第一层粗筛准则。它能够显著缩小搜索空间，并且筛出的候选具有可解释的几何特征；但它仍不是最终任务性能指标，不能替代后续 d' 精筛和泊松噪声重建验证。

## 8. 阶段性结论

本实验的结论不应理解为 EIG 已经给出最终任务最优架构，而应理解为：EIG 在三维参数化架构族上能够产生非平坦的排序，并且 top 候选在层数、层高、角偏移和朝向插值等几何参数上呈现可观察的分布偏好。

因此，EIG 可以作为后续三维光源展布搜索的第一层粗筛准则：先删除低信息价值候选，把搜索空间缩小到 top 5% 到 10%，再进入 d' 精筛和泊松重建验证。
